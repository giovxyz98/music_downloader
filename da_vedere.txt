SEZIONE PACKAGING — DA FARE SOLO ALLA FINE DEGLI SVILUPPI
════════════════════════════════════════════════════════════

A. Ottimizzazione import → from X import Y

  Attualmente il file usa import interi (es. import re,
  import json, import threading). PyInstaller include
  l'intero modulo anche se ne usi solo una funzione,
  gonfiando l'exe inutilmente.

  Soluzione: sostituire tutti gli import con from X import Y
  in modo da includere solo ciò che viene effettivamente usato.

  Esempi:
    import re                    →  from re import sub, compile
    import os                    →  from os import startfile, makedirs
    import json                  →  from json import load, dump
    import threading             →  from threading import Thread, Lock
    import urllib.request        →  from urllib.request import urlopen
    from pathlib import Path     →  invariato (già corretto)
    from concurrent.futures ...  →  invariato (già corretto)

  Da fare prima di eseguire PyInstaller, non prima.
  Non cambia il comportamento del programma.

B. Packaging dell'app → MusicDownloader.exe

Questo progetto è perfetto per diventare app desktop reale.

Serve solo pyinstaller per creare MusicDownloader.exe.
Molti utenti non installano Python.

Attenzione: yt-dlp e ffmpeg richiedono gestione speciale
in PyInstaller (binari aggiuntivi, hook). Non è banale su
Windows. Usare --onefile --windowed e includere ffmpeg.exe
nella cartella.
RISCHIO: alto se fatto senza esperienza con PyInstaller.
